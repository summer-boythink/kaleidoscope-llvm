# Kaleidoscope 编译器：从文本到机器码的奇妙旅程

> 想象一座工厂：原材料（源代码）从大门进入，经过一道道流水线，最终变成闪闪发光的产品（机器码）。
> 这个项目就是这座工厂的蓝图。

---

## 全景地图

```
你输入的文字
    │
    ▼
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌───────────┐    ┌─────────┐
│  词法分析  │───▶│  语法分析  │───▶│  代码生成  │───▶│  优化器     │───▶│  JIT执行  │
│  Lexer   │    │  Parser  │    │  CodeGen  │    │  Optimizer │    │  JIT    │
└─────────┘    └─────────┘    └─────────┘    └───────────┘    └─────────┘
  切碎文字        搭建骨架        翻译方言        打磨抛光        开动机器
```

每个模块各司其职，依次接力，把一行人类可读的文本变成 CPU 真正执行的指令。

## 1. 词法分析器 —— 文字的"切片厨师"

**文件**: [lexer.hpp](../include/lexer.hpp) / [lexer.cpp](../src/lexer.cpp)

想象一位寿司师傅，把一整条鱼切成一片片整齐的鱼片。词法分析器做的事情一样——把连续的字符流切成一个个有意义的"词元"（Token）。

### 它如何工作

```
输入: "def add(x y) x + y"

词法分析器逐字符扫描:
  d-e-f        → 识别出关键词，吐出 Token::def
  空格         → 跳过，就像厨师去掉鱼骨
  a-d-d        → 识别出标识符，吐出 Token::identifier ("add")
  (            → 原样吐出 Token::'('  (ASCII值 40)
  x            → 标识符 Token::identifier ("x")
  ...
  +            → 运算符 Token::'+'  (ASCII值 43)
```

### Token 的编码规则（非常巧妙！）

```
Token 值域:
  -1 ~ -13   → 特殊词汇（关键词、标识符、数字等）
   0 ~ 255   → 单字符运算符（直接用 ASCII 码值）

这就像一套双拼输入法：
  - 负数区间 = "短语词库"（def、extern、if...）
  - 正数区间 = "单字直出"（+、-、*、/、(、)...）
```

### 能识别的"食材"清单

| 食材类型 | 例子 | Token |
|---------|------|-------|
| 数字 | `3.14` | `tok_number` |
| 标识符 | `foo`, `x`, `add` | `tok_identifier` |
| 关键词 | `def`, `extern`, `if`, `for` | 对应的 `tok_*` |
| 运算符 | `+`, `-`, `*`, `/`, `<` | ASCII 值本身 |
| 注释 | `# 这是注释` | 整行跳过，像鱼刺一样被丢弃 |

## 2. 语法分析器 —— 语言的"建筑师"

**文件**: [parser.hpp](../include/parser.hpp) / [parser.cpp](../src/parser.cpp)

如果词法分析器是切片厨师，那语法分析器就是建筑师——他把一片片鱼片搭成寿司塔，把一个个 Token 搭成抽象语法树（AST）。

### 运算符优先级 —— 数学的"社会等级"

```
优先级就是"谁先算"的规矩，就像排队：

  <*> 和 </>  ──── 40分 ────  老大，先算我！
  <+> 和 <->  ──── 20分 ────  老二，轮到我了
  <<>        ──── 10分 ────  小弟，我最后...

所以 1 + 2 * 3 的解析过程：
  ┌─ BinaryExpr('+') ─┐
  │                    │
  NumberExpr(1)   ┌─ BinaryExpr('*') ─┐
                  │                    │
              NumberExpr(2)       NumberExpr(3)

  先把 * 绑紧，再把 + 绑上去——就像先穿内衣再穿外套
```

### Pratt 解析法 —— 一种优雅的"左右逢源"

Parser 使用 **Pratt 解析**（也叫算子优先解析），核心思想是：

```
ParseExpression():
  1. 先解析左边的东西（primary）── 比如一个数字或变量
  2. 然后看右边有没有运算符
  3. 如果运算符优先级"够格"，就把它拉过来组合
  4. 递归处理，直到运算符优先级不够为止

这就像搭积木：
  1 + 2 * 3
  └─先拿"1"
     看到"+"，优先级20，OK我接受
     └─先拿"2"
        看到"*"，优先级40，比+高，我抢！
        └─先拿"3"
           没有更高优先级的了，收工
        "2*3"打包完成
     "1+(2*3)"打包完成
```

### AST 节点族谱

**文件**: [ast.hpp](../include/ast.hpp)

```
                    ASTNode (老祖宗，虚基类)
                        │
          ┌─────────────┴─────────────┐
          │                           │
     ExprAST                      PrototypeAST
     (表达式)                     (函数签名)
          │                           │
    ┌─────┼─────┬──────┬──────┐       │
    │     │     │      │      │       │
  Number  Var  Binary Unary  Call   FunctionAST
  (数字) (变量) (二元) (一元) (调用)  (完整函数 = 签名 + 函数体)
```

每个节点就像一个乐高积木块：

- **NumberExprAST** —— 一块数字积木，写着 `3.14`
- **VariableExprAST** —— 一块名牌积木，标着变量名 `x`
- **BinaryExprAST** —— 一块双孔积木，左边插 `1`，右边插 `2`，中间写 `+`
- **CallExprAST** —— 一块多孔积木，上面写着函数名，各个孔插参数

## 3. 代码生成器 —— LLVM 的"同声传译"

**文件**: [codegen.hpp](../include/codegen.hpp) / [codegen.cpp](../src/codegen.cpp)

AST 是"概念图"，CPU 看不懂。代码生成器就像一位同声传译员，把 AST 翻译成 LLVM IR——一种接近机器码但人类仍可读懂的中间语言。

### 核心翻译规则

```
Kaleidoscope 世界的万物都是 double（64位浮点数）。
这是一个"万物皆数"的哲学世界。

翻译对照表:
┌──────────────────────┬──────────────────────────────────┐
│  Kaleidoscope        │  LLVM IR                         │
├──────────────────────┼──────────────────────────────────┤
│  数字 3.14           │  ConstantFP(3.14)                │
│  变量 x              │  NamedValues["x"] → alloca/load  │
│  a + b               │  CreateFAdd(a, b)                │
│  a - b               │  CreateFSub(a, b)                │
│  a * b               │  CreateFMul(a, b)                │
│  a / b               │  CreateFDiv(a, b)                │
│  a < b               │  CreateFCmpULT → UIToFP          │
│  foo(a, b)           │  CallInst @foo(a, b)             │
│  def f(x) x+1        │  define double @f(double %x)     │
└──────────────────────┴──────────────────────────────────┘
```

### `<` 运算符的"身份转换"小剧场

```
Kaleidoscope 里，万物皆 double。
但比较运算的结果是 bool（0 或 1）。

所以 "<" 需要一个"变身"过程：

  a < b
    │
    ▼  CreateFCmpULT
  bool (i1)  ←  比较结果是布尔值，但Kaleidoscope不认这个！
    │
    ▼  CreateUIToFP
  double (0.0 或 1.0)  ←  转回浮点数，完美融入Kaleidoscope世界

就像一个留洋归来的人：
  在国外说英语（bool），回到家乡得说方言（double）
```

### JIT 执行 —— 即时编译，当场出结果

```cpp
executeTopLevelExpr():
  1. 把裸表达式包进一个匿名函数 "__anon_expr()"
  2. 编译成 LLVM IR
  3. 优化
  4. JIT 编译成机器码
  5. 执行！
  6. 返回结果

就像在餐厅点单：
  你说 "1+2"  →  厨房立刻做  →  端上桌 "= 3"
  不需要打包带走，当场吃完！
```

## 4. 优化器 —— 代码的"美容师"

**文件**: [optimizer.hpp](../include/optimizer.hpp) / [optimizer.cpp](../src/optimizer.cpp)

优化器就像一位美容师，把粗糙的 LLVM IR 打磨得光滑亮丽——运行更快、体积更小。

### 优化级别 —— 从"素颜"到"精修"

```
O0  素颜模式     →  什么都不做，保留原汁原味（调试用）
O1  基础护理     →  去死皮(死代码) + 补水(指令合并) + 瘦脸(简化CFG)
O2  精致美颜     →  O1全套 + 升阶(mem2reg) + 换位(重结合) + 去重(GVN)
O3  换头级别     →  O2多来几遍，能挤的优化全挤出来
```

### 各个优化 Pass 的生动解释

```
┌─────────────────────────────────────────────────────────┐
│  PromotePass (mem2reg)  —— "寄存器晋升仪式"              │
│                                                         │
│  优化前: 变量住在内存（栈）上，每次读写要 load/store      │
│  优化后: 变量晋升到寄存器（SSA值），直接传递，零延迟       │
│                                                         │
│  就像: 小职员(memory)被提拔为高管(register)               │
│        以前汇报要层层审批(load/store)，现在直达天听(SSA)   │
├─────────────────────────────────────────────────────────┤
│  InstCombinePass  —— "指令合并术"                        │
│                                                         │
│  优化前: x + 0;  y * 1;  z / 1                         │
│  优化后: x;       y;       z                            │
│                                                         │
│  就像: "请问您的姓名是？" → "您贵姓？"（精简冗余表达）     │
├─────────────────────────────────────────────────────────┤
│  ReassociatePass  —— "重结合术"                          │
│                                                         │
│  优化前: (a + 1) + 2                                    │
│  优化后: a + 3                                          │
│                                                         │
│  就像: 先去A站再去B站 → 直达C站，少走弯路                │
├─────────────────────────────────────────────────────────┤
│  GVNPass  —— "全局值编号"（去重神器）                    │
│                                                         │
│  优化前: a = x + y;  b = x + y;  (算了两遍)             │
│  优化后: a = x + y;  b = a;     (复用第一次的结果)       │
│                                                         │
│  就像: 你问了两遍同一个问题，我第二次直接说"同上"         │
├─────────────────────────────────────────────────────────┤
│  SimplifyCFGPass  —— "控制流瘦身"                        │
│                                                         │
│  优化前: 一个空的基本块，只跳转到下一个块                  │
│  优化后: 直接合并，去掉无意义的跳转                       │
│                                                         │
│  就像: 去掉一个只会说"请到下一窗口"的空窗口               │
├─────────────────────────────────────────────────────────┤
│  DCEPass  —— "死代码消除"                                │
│                                                         │
│  优化前: x = 1;  y = 2;  return x;  (y 没人用)          │
│  优化后: x = 1;  return x;        (y 被删除)            │
│                                                         │
│  就像: 清理房间里没人穿的旧衣服，释放空间                 │
└─────────────────────────────────────────────────────────┘
```

### 优化的实际效果

```
输入: 1 + 2 * 3

优化前的 IR:
  %1 = fmul double 2.000000e+00, 3.000000e+00
  %2 = fadd double 1.000000e+00, %1
  ret double %2

优化后的 IR:
  ret double 7.000000e+00

美容师把三行缩减成一行！2*3=6, 1+6=7，直接算出来了。
这就是"常量折叠"——编译期就算完，运行时白嫖。
```

## 5. JIT 编译器 —— 代码的"瞬间实化术"

**文件**: [jit.hpp](../include/jit.hpp) / [jit.cpp](../src/jit.cpp)

JIT（Just-In-Time）编译器是整个系统的灵魂。它让代码"从纸上跳到现实"——不需要提前编译成文件，直接在内存中编译并执行。

### LLVM ORC LLJIT —— 现代魔法引擎

```
传统编译（AOT）:
  源码 → .o文件 → 链接 → 可执行文件 → 运行
  就像: 写好菜谱 → 出版成书 → 读者买书 → 照着做菜

JIT编译:
  源码 → LLVM IR → 机器码(在内存中) → 直接执行
  就像: 厨师站在你旁边，你说什么菜他当场做当场给你吃
```

### JIT 的关键能力

```
┌──────────────────────────────────────────────────────────────┐
│                                                              │
│  1. addModule() —— "知识注入"                                 │
│     把编译好的 IR 模块加入 JIT，就像给厨师一本新菜谱           │
│                                                              │
│  2. lookup() —— "召唤术"                                     │
│     根据函数名找到机器码地址，就像按菜名找到对应菜品           │
│                                                              │
│  3. ResourceTracker —— "记忆管理"                             │
│     匿名表达式用完就删，像吃自助餐：                          │
│       每次点新的之前，先把上一盘剩菜收走                       │
│     用户定义的函数则永久保留，像你的拿手菜，一直在菜单上       │
│                                                              │
│  4. DynamicLibrarySearchGenerator —— "外挂技能"               │
│     允许调用 C 标准库函数（sin, cos, printf...）              │
│     就像厨师不会做每道菜，但可以打电话叫外卖（调用外部函数）    │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 6. 主程序 —— 交互式 REPL

**文件**: [main.cpp](../src/main.cpp)

REPL（Read-Eval-Print Loop）是一个交互式循环，就像和编译器聊天：

```
┌──────────────────────────────────────────────────────┐
│                     主循环                            │
│                                                      │
│   ┌─── ready> ────┐                                  │
│   │               │                                  │
│   │  读取输入     │                                  │
│   │      │        │                                  │
│   │      ▼        │                                  │
│   │  词法分析     │                                  │
│   │      │        │                                  │
│   │      ▼        │                                  │
│   │  语法分析     │                                  │
│   │      │        │                                  │
│   │      ▼        │                                  │
│   │  ┌─ def? ─────┤─── 定义函数 ──▶ 生成IR → 优化    │
│   │  │            │                                  │
│   │  ├─ extern? ──┤─── 声明外部函数                   │
│   │  │            │                                  │
│   │  └─ 其他 ─────┤─── 当作表达式 ──▶ JIT执行 → 打印  │
│   │               │                                  │
│   │  ◀────────────┘                                  │
│   └─────────────────┘                                │
└──────────────────────────────────────────────────────┘
```

### 一次完整的对话

```
ready> def fib(x) if x < 3 then 1 else fib(x-1) + fib(x-2);
         │       │          │       │                    │
         def    签名        条件    then分支             else分支
         │
         ▼
    Parser 生成 AST
         │
         ▼
    CodeGen 生成 LLVM IR
         │
         ▼
    Optimizer 优化
         │
         ▼
    JIT 编译并缓存
         │
         ▼
    "Defined function: fib"

ready> fib(10);
         │
         ▼
    JIT 查找 fib 函数
         │
         ▼
    执行！递归计算
         │
         ▼
    = 55
```

## 数据流全景图

```
"1 + 2 * 3"
     │
     │ Lexer (切词)
     ▼
[tok_number:1] [tok_'+'] [tok_number:2] [tok'*'] [tok_number:3]
     │
     │ Parser (搭树)
     ▼
        BinaryExpr('+')
       /              \
  NumberExpr(1)   BinaryExpr('*')
                 /              \
           NumberExpr(2)    NumberExpr(3)
     │
     │ CodeGen (翻译)
     ▼
  %1 = fmul double 2.0, 3.0
  %2 = fadd double 1.0, %1
  ret double %2
     │
     │ Optimizer (精修)
     ▼
  ret double 7.0
     │
     │ JIT (实化执行)
     ▼
  = 7.0
```
